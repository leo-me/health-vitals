## 🎓 面试常见问题 & 回答框架

### Q1: "为什么选择 FastAPI？"
```
A: "我需要一个高性能、易于使用的框架。
   FastAPI 基于 Starlette，支持异步处理，
   API 文档自动生成，开发效率高。
   性能接近 Go/Node.js，非常适合数据密集型应用。"
```

### Q2: "如何处理实时数据？"
```
A: "有两种方式：
   1. WebSocket（完全实时，但增加复杂性）
   2. HTTP polling + 自动刷新（简单可靠）
   
   在这个项目中我选择了 polling，
   因为我的目标是展示 full-stack 能力，
   而不是 WebSocket 的复杂性。"
```

### Q3: "数据量特别大时怎么办？"
```
A: "会做几个优化：
   1. 数据库：加索引、分片、物化视图
   2. 缓存：Redis 缓存热数据
   3. 异步：用 Celery 处理长耗时任务
   4. 监控：CloudWatch 告警"
```

### Q4: "安全方面怎么考虑的？"
```
A: "几个层面：
   1. 认证：JWT token（签名验证）
   2. 授权：Role-based access control (RBAC)
   3. 数据：HTTPS、加密敏感字段
   4. 注入：Pydantic 数据验证、SQL 参数化
   5. 日志：敏感信息不记录"
```
